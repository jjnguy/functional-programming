Language Paradigms

  • Declarative
  • Imperative/Procedural
  • Object Oriented
  • Functional

Declarative

Tell a computer 'what' to do, not 'how' to do it

SQL


							SELECT something FROM somewhere WHERE it looks like this

C#


							collection.Where(it => it.IsMatch).OrderBy(it => it.Name)

Imperative/Procedural

Ordered statments that change a program's state to a desired result

C (C++, C#, Java, Javascript, etc)


								int a = 6;
								int a2 = a * 2;

Object Oriented

Encapsulate logic and data into 'classes'

Ruby


class Incrementor
  @value = 0
  def increment
    @value = @value + 1
  end
  def value
    return @value
  end
end

Functional

Utilizes functions to contain state and logic.

Lisp


              (defun fatorial (n)
                (cond
                  ((= n 1) 1)
                  (t (* n (fatorial (- n 1))))))

Multi-Paradigm Languages

Languages that allow for multiple paradigms.

C#


              // procedural
              var numbers = new List<int> { 0, 1, 2, ... 100 };
              var even = new List<int>();
              foreach (var num in numbers) 
              { 
                if (num % 2 == 0) even.Add(num); 
              }
            

              // functional/declarative
              var numbers = new List<int> { 0, 1, 2, ... 100 };
              var even = numbers.Where(num => num % 2 == 0).ToList();

Javascript


              let numbers = [0, 1, 2, ... 100];
              let even = [];
              for (var ix = 0; ix < numbers.length; ix++) {
                if (numbers[ix] % 2 == 0) even.push(numbers[ix]);
              }
            

              let numbers = [0, 1, 2, ... 100];
              let even = numbers.filter(num => num % 2 == 0);

Why Functional?

  • Stateless
  • Intentional

Stateless

Programmers are not as smart as we'd like to think

Limit scope of info a programmer needs to keep in their head


              let books = [{ title: '...', author: '...'}, ... ];
              let grouped = {};
              for(var ix = 0; ix < books.length; ix++) {
                let book = books[ix];
                if (book.author[0] == 'a') {
                  if (!grouped[book.author]) grouped[book.author] = [];
                  grouped[book.author].push(book);
                }
              }
            

              let books = [{ title: '...', author: '...'}, ... ];
              let grouped = books
                .filter(book => book.author[0] == 'a')
                .groupBy(book => book.author);
            

              // both result in objects that look like:
              {
                "_author name_": [ { book by corresponding author }, ... ]
              }

How is the Second Example Stateless?

  • Intermediate variables
  • Partial results
  • Implementation details

Intentional

Code that declares its intent serves 2 purposes - computation and documentation

How is the Second Example Intentional?

  • Separate Steps
  • Named Steps
  • No Implementation Details
